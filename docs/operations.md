# Эксплуатация и будущие доработки (шаг 12)

## Эксплуатационные сценарии
- Пример правила «VIP open gate»: белый список + `trigger_relay` + `send_webhook`.
- Журнал аудита: храните `/api/v1/auth/me` + действие в логах JSON (уровень INFO/DEBUG по задачам расследования).
- Экспорт событий: фильтры `channel_id`, `plate`, период; форматы CSV/JSON/PDF (генераторы отчетов на стороне UI/API).

## Отчёты и экспорт
- CSV: выгрузка таблицы событий с полями `timestamp, channel_id, plate, confidence, list_type`.
- JSON: полная структура события (`bbox`, `track_id`, `meta`).
- PDF: сборка из шаблона с агрегатами по каналам и временным диапазонам.

## Рекомендации по будущим улучшениям
- ReID: добавление марки/цвета авто, сохранение в `meta` и фильтрация в поиске.
- Двухстрочные номера: расширение шаблонов стран и постобработки на несколько строк.
- PTZ: интеграция с ONVIF PTZ для автоповорота камеры по событию.
- Edge-режим: оффлайн-буферизация событий и синхронизация при восстановлении связи.
